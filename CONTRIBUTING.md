# Contributing to ADK Skill Patterns

Thank you for your interest in contributing! This document provides guidelines for contributing to the ADK Skill Patterns project.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists in the [issue tracker](https://github.com/MaxLaurieHutchinson/adk-skill-patterns/issues)
2. If not, create a new issue with:
   - Clear title and description
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Screenshots or examples if applicable

### Submitting Changes

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b feature/amazing-improvement`)
3. **Make your changes** following the guidelines below
4. **Test** your changes thoroughly
5. **Commit** with clear messages (`git commit -m 'Add: new pattern for X'`)
6. **Push** to your fork (`git push origin feature/amazing-improvement`)
7. **Open a Pull Request** with a detailed description

### Contribution Guidelines

#### For New Patterns

- Must solve a distinct problem not covered by existing patterns
- Include a clear use case and example
- Provide a working SKILL.md template
- Explain when NOT to use the pattern

#### For Pattern Examples

- Real-world use cases preferred
- Include complete, working SKILL.md
- Show before/after comparison if applicable

#### For Documentation

- Keep language clear and concise
- Use examples to illustrate concepts
- Update the decision tree if adding patterns

### Code Style

- Use YAML frontmatter for SKILL.md metadata
- Follow existing formatting in templates
- Use semantic line breaks in markdown
- Keep lines under 100 characters where possible

### Commit Message Format

```
Type: Brief description

Detailed explanation if needed.

- Bullet points for changes
- Reference issues: #123
```

Types:
- `Add:` New feature, pattern, or example
- `Fix:` Bug fix
- `Docs:` Documentation changes
- `Refactor:` Code restructuring
- `Test:` Adding tests

### Review Process

1. All contributions require review
2. Address feedback promptly
3. Keep discussions focused and constructive
4. Be respectful of reviewers' time

### Recognition

Contributors will be acknowledged in the README.md file.

## Questions?

- Open a [Discussion](https://github.com/MaxLaurieHutchinson/adk-skill-patterns/discussions)
- Contact via [ClawHub](https://clawhub.ai/skills/adk-skill-patterns)

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Accept different viewpoints and experiences
- Prioritize community benefit

Thank you for helping make agent skills more reliable and composable! 🧩
